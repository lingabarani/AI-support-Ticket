# AI-Powered Customer Experience & Business Intelligence Platform

## Tech Stack
- **Frontend:** React.js + Vite + Tailwind CSS + Recharts + Framer Motion
- **Backend:** Node.js + Express.js + MongoDB + JWT Auth
- **AI:** Amazon Bedrock (Claude)
- **Analytics:** Amazon QuickSight embed placeholders
- **Storage:** Amazon S3
- **Infra:** Terraform for AWS resources

---


## Setup

### Frontend
```bash
cd aisupport-frontend
npm install
npm run dev        # http://localhost:5173
npm run build      # Production build
```

### Backend
```bash
cd aisupport-backend
cp .env.example .env   # Fill in your credentials
npm install
npm run dev            # http://localhost:5000
```

### AWS Terraform
```bash
cd aisupport-backend/terraform
copy terraform.tfvars.example terraform.tfvars
terraform init
terraform plan
terraform apply
```

The Terraform stack is designed for a personal AWS account and intentionally does not create Lambda or CloudWatch resources. It creates:

- Two private S3 buckets for raw tickets and analytics output
- One Amazon Bedrock Agent and alias for the AI support assistant
- One IAM policy for the backend runtime to use S3, Bedrock, and QuickSight embed APIs

After `terraform apply`, copy these outputs into `aisupport-backend/.env`:

```text
AWS_ACCOUNT_ID=<aws_account_id>
AWS_REGION=<aws_region>
RAW_BUCKET=<raw_bucket_name>
ANALYTICS_BUCKET=<analytics_bucket_name>
BEDROCK_MODEL_ID=<bedrock_model_id>
BEDROCK_AGENT_ID=<bedrock_agent_id>
BEDROCK_AGENT_ALIAS_ID=<bedrock_agent_alias_id>
```

Keep AWS access keys in your local AWS CLI config or terminal environment. Do not commit them to `.env`, `.tfvars`, or source files.

## Amazon QuickSight Integration
- QuickSight embed URLs are generated by `GET /api/quicksight/embed-url?role=<role>`.
- Configure `QUICKSIGHT_USER_ARN` and the `QS_DASHBOARD_*` dashboard IDs in `aisupport-backend/.env`.
- The Terraform output `backend_runtime_policy_arn` is the policy to attach to the IAM user/role that runs this backend.

## Deployment
- **Frontend:** Deploy `aisupport-frontend/dist/` to Vercel, Netlify, or AWS CloudFront/S3.
- **Backend:** Deploy `aisupport-backend` to Render, Railway, Fly.io, or AWS ECS.
- **DB:** Use MongoDB Atlas and set `MONGODB_URI` in the backend environment.
